    义工平台原型和功能结构

    2024-11-06 Atom.X

    依据前文多个义工平台的案例，抽取其通用的原型/模型，描述平台结构如下。

    用户类型 #

    平台用户分为两种，自然人people和单位community。

    个人 #

    个人People等同于客人guest，是自然人个体，在海外平台都需要付费注册。

    • 职能

    按照与主人Host预先的约定，提供以下服务。

    工作3-4hrs/d

    年轻活力思维

    专业知识技能创意

    陪伴对话交流协作

    • 基本信息

    昵称

    logo

    邮箱

    密码

    • 实名认证

    真实姓名（文本）

    真实头像

    性别（单选）

    生日（日期单选）

    国籍（单选/多选）

    护照/身份证图片/编号（图片/数字）

    常住地址（文本）

    手机/座机（数字）

    常用语言（多选）

    自我介绍（文本框）

    用户协议（勾选框）

    • 身份标签

    People自然人可以绑定到一个Community单位中，形成所属关系，每个People人将打一个身份标签，类型：

    单位主管（负责人、法定代表人），证明与单位关系后，具备给任何自然人打身份标签的权限；

    单位员工（正职，实习生），可以显示所归属的单位，经过其本人同意，才能显示身份标签；

    志愿者（短期服务中……），可以显示所述的单位，经过其本人同意，才能显示身份标签；

    尚没有加入任何一家单位（短期服务）的人，没有任何标签，就是个体的自然人，可以转化为志愿者。

    单位 #

    单位Community在大多等同于主人host，单位是法人主体，持有营业执照，许多人集合的共同体组织，在海外平台通常免费注册。

    • Host 主人职能

    按照预先的约定，提供以下服务。

    吃饭/住宿

    业余休闲

    就业创业

    用户类型可以综合多样化，包括农场/ 工厂/ 旅舍/社区/ 学校/ 寺庙/ 公益组织等等。

    • 基本信息

    简称

    logo

    邮箱

    密码

    • 实名认证

    单位全称（文本）

    单位类型（多选，例如：公司、农场、学校、工坊、餐厅、旅馆、社区- 村委会、公益组织）

    创建日期（日期单选）

    国籍（单选）

    营业执照图片/编号（图片/数字）

    住所地址（文本）

    手机/座机（数字）

    单位简介（文本框）

    用户协议（勾选框）

    • 单位联合

    当一家单位有多家下属单位时，需要将多个单位关系绑定为联合体，例如：

    某家公司开设多个农场，在不同地址，属于同一家公司经营；

    • 招募公告

    单位可以发布各种招募公告，招募志愿者，实习生员工等信息。

    会员付费 #

    目前按照国际主流模式，由Guest（志愿者）支付年度会员费，不同国家地区的价格不同，由于中国用户众多且市场庞大，可以考虑从农场主一方收费，或者从双方两头各收一半，或完全取消收费，在用户数量达到一定程度，通过电商导购和广告等方式营收。

    会员类型包括个人和单位 Community，支付方式满足国际或不同来源用户的支付习惯。

    • 支付宝
    • 微信
    • 银联
    • Paypal
    • 信用卡

    用户权限 #

    用户注册申请，通过审核后，将获得相应的用户权限。

    基本权限 #

    填写了“基本信息”的用户

    可以翻阅有限的host和guest列表；

    可以翻阅限量的网络评论和图片；

    中级权限 #

    通过“实名认证”的用户

    可以翻阅全部的host和guest列表

    可以翻阅全部的网络评论和图片；

    高级权限 #

    “付费用户”将有最高权限

    在实名认证的权限之上，host和guest双方可以发送站内信对话，可以查看对方的联系方式，包括邮箱，电话，IM等；

    最高权限用户才可以使用“身份标签”功能，包括community单位和people个人。

    网站首页 #

    焦点图 #

    提供3-5张精选主图，或者背景视频

    使用指南 #

    通过图文非常简捷指引用户

    Community单位3步导引成功注册

    People个人3步导引成功注册

    单位搜索 #

    搜索框提供以下几种检索方法：

    单位名称，

    地区，

    行业类型

    检索列表呈现：

    单位名称

    地址

    行业类型

    地图 #

    地图提供按照地区选择单位

    动态更新 #

    鼓励用户UGC更多贡献内容，可以将个人或单位的社交图文空间绑定facebook/twitter/微信/微博等社交媒体，实时通过社交媒体与该平台同步分享社交媒体信息。

    动态更新是高级权限，包括单位和个人，需要完成实名认证和会员付费才可以看到。

    导航菜单 #

    提供以下选项：

    返回首页

    查看单位

    动态更新

    关于我们（团队介绍、联系客服）

    登陆/注册

    帮助（FAQ）

    附加功能 #

    在以上基础功能开发出来之后，平台运作一段时间后，根据实际情况评估，再决定是否要开发附加功能。主要包括如下：

    信用评级 #

    类似于升级版的大众点评，淘宝点评，可以允许志愿者和公众对该单位进行最高不超过5星的评级，以及留下文字点评；

    例如农场的评级结构包括：生产环境，生产方式，生活环境，农场主

    电商团购 #

    单位可以发布团购信息，按照参团人数和购买量设置商品阶梯价，以及给电商平台的导购链接。

    活动发布 #

    各种类型的单位可以发布周末活动，或各种常态化活动，例如农场可以发布“开心地主”活动，召集市民周末到农场参加各类亲子家庭活动。

    广告链接 #

    链接CPS，CPC付费广告平台，给用户做精准推荐，在足够大流量情况下将成为一个重要营收来源。

    市场前景 #

    • 用户数量

    根据2017年12月14日发布，第三次全国农业普查主要数据公报，截止2016年12月31日，涉农类单位数量：

    合作社179万

    农业经营单位204万

    农业经营户20743万

    （规模户398万）

    乡镇31925个

    自然村317万

    村委会556264个

    涉农居委会40186个

    总量＞1亿

    eg. 仅仅规模农业经营单位＞700万

    • 网络流量

    当用户规模达到百万级，可以实现UV＞千万级， PV＞亿级

    • 营收规模

    若解决每家规模农业经营单位1位/年的人力资源，则年需求量达700万人次，按照10元/年会员费计算，将有7000万/年营收。由于该平台定位为公益，社会企业，民非类型，可以获取公益基金，以及政策相应支持。如果不考虑收会员费以及任何商业营收，探索通过政府和公益基金会也可以获取足够保障。

    投入产出 #

    假设一个成功的完整产品生命周期为10

    • 首年从0-1阶段

    投入：

    主要的人力资源投入各1名如下，

    产品经理，

    UED，

    前端，

    后端，

    运营，

    按照目前江浙沪地区的市价15万/年，总计需要约75万，以及其他各种费用，首年保守估计需要100万开销。

    产出：

    首年做到1万名用户，按照100元/年会员费计算，则可以达到年营收100万，基本维持收支平衡。但目前不确定，做到1万个用户的其他不确定性的额外成本。

    • 第二年从1-3阶段

    投入：

    在以上人力资源基础上，还要增设岗位各1名如下，

    行政（法务、财务），

    市场，

    客服，

    IT开发1-2名，

    第二年人力资源总计可能超过10位，需要200万开销。这个阶段需要市场推广，运营费用增加，保守估计需要300万/年总支出。

    产出：

    第二年做到30万名用户，按照10元/年会员费计算，则可以达到年营收300万，基本维持收支平衡。但目前不确定，做到30万用户的其他额外开支，以及不确定性障碍。该阶段可以考虑吸纳一些种子投资。

    • 第三年从4-8阶段

    投入：

    在以上人力资源基础上，还要增设岗位各2名如下，

    运营，

    客服，

    IT开发，

    第三年人力资源总计可能超过15位，需要300万开销。该阶段需要市场推广，运营费用倍增，保守估计需要600万/年总支出。

    产出：

    第三年做到100万名用户，按照10元/年会员费计算，则可以达到年营收1000万，已经实现盈利，而且全站流量超千万人次，具有电商和网络广告潜力，但考虑其公益属性区别于常规商业，需要开发一些创新的营收方式。但目前不确定，做到100万用户的其他额外开支，以及不确定性障碍。该阶段可以考虑吸纳pre A轮投资。
